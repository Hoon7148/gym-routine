# gym-routine 디자인 브리프

## 앱 한 줄 정의
선수 유튜브 루틴의 **뼈대(종목+순서)**만 큐레이션해 보여주고, 세트·무게·폼 같은 디테일은
영상 타임스탬프 딥링크로 유도하는 운동 기록 앱. "오늘 뭐 할지 고민하는 결정 피로"를 없애는 게 목적.

## 타겟 사용자
5년 이상 웨이트 트레이닝을 해온 경력자. 초보자용 설명은 필요 없고, 운동 구조만 알면
스스로 세트·무게를 커스텀할 수 있는 사람. 유튜브에서 선수 루틴을 찾아보는 데 시간을
쓰는 걸 귀찮아함.

## 디자인 톤앤매너
- **플랫, 미니멀.** 그라데이션·그림자·장식 없음. 헬스장 특유의 무게감 있는 분위기.
- **다크 베이스** (기본: 거의 검정에 가까운 배경, 밝은 텍스트) — 헬스장/운동 앱 관습에 맞춤.
- 정보 밀도는 낮게. 화면당 할 일이 명확해야 함(선택 피로 자체가 이 앱이 없애려는 문제).
- 숫자(무게·횟수·인원수)가 자주 등장하므로 숫자 가독성이 중요 — 굵기·크기로 위계 표현.
- 아이콘은 아웃라인 스타일, 장식이 아니라 기능 표시용으로만.

## 기술 제약
- Mobile-first PWA (React 19 + Vite + Tailwind v4). 데스크톱 고려 안 함.
- 뷰포트 기준 375~430px.
- 컴포넌트: 카드, 배지/필, 진행바, 하단 탭바, 인풋(무게/횟수), 체크리스트.

## 핵심 화면 5개

### 1. 홈
- 상단: "오늘 뭐 할까요?" + 날짜
- 히어로 카드: "오늘의 추천 루틴" — 영상 썸네일 + 재생 버튼 오버레이 + 재생시간 배지,
  "N명이 담음"(사회적 증거), CTA "내 운동으로 가져오기" + 새로고침(다른 추천) 버튼
- 개인 넛지 배너: "하체 운동한 지 N일 됐어요" 같은 경고색 알림 바
- "오늘 많이 담긴 루틴" 랭킹 리스트 (순위·썸네일·담은 인원수)
- "오늘 인기 종목" 태그 칩 리스트
- 하단 탭바: 홈 / 탐색 / 기록 / 내정보

### 2. 루틴 상세 (핵심 화면)
- 영상 썸네일 + 재생시간
- 선수 아바타 + 이름 + "원본 영상 보기" 링크
- **운동 순서 목차** — 번호 매긴 리스트, 각 항목에 종목명 + "▶ 타임스탬프" 배지(탭하면
  영상의 그 구간으로 딥링크)
- 하단 안내: "세트·무게·자세 팁은 영상에서 확인하세요 — 이 앱은 운동 뼈대만 알려줍니다"
- CTA: "내 운동으로 가져오기" + 북마크 버튼
- **의도적으로 세트/무게 수치는 안 보여줌** — 이게 이 앱의 핵심 차별점

### 3. 오늘 운동 기록
- 진행률 바 (N/전체 완료, 경과 시간)
- 완료된 종목: 초록색, 취소선 처리된 요약 한 줄로 축소
- 진행 중 종목: 강조 테두리, "▶ 영상 구간 다시보기" 버튼, "지난번: Xkg × Y회" 힌트,
  세트별 무게/횟수 입력칸(지난 기록이 회색 placeholder로 미리 채워짐) + 완료 체크
- 아직 안 한 종목: 흐리게, "첫 시도"/"지난번 X" 배지만

### 4. 큐레이터 태깅 도구 (내부 관리용, 사용자용 아님)
- 유튜브 URL 입력 → 자막 로드 상태 표시
- AI가 추출한 종목+대략 타임스탬프 목록 (회색 "AI 추정" 상태)
- 각 항목: ±초 조정 버튼, "확정" 버튼 → 확정되면 초록색으로 전환
- 진행률 "N/전체 확정"
- 하단: 임시저장 / 루틴 발행 버튼

### 5. 큐레이션 콜드스타트 폴백
- 유저 수가 적을 때는 "N명이 담음" 대신 큐레이터가 고른 "이주의 루틴" 배지로 대체 —
  같은 레이아웃에서 데이터만 전환되는 구조여야 함.

## 컬러/상태 의미
- 초록 = 완료/검증됨
- 노랑/주황 = 개인 리마인더(오래 안 한 부위)
- 회색 = 아직 미확정(AI 추정, 미시작 종목)
- 강조색(파랑 계열) = 오늘의 추천, CTA, 딥링크 배지

## 요청하고 싶은 것
위 5개 화면(특히 2번 루틴 상세, 3번 기록 화면)을 실제 컴포넌트 단위로 나눠 UI 디자인
시안을 만들어줘. Tailwind 유틸리티 클래스 기반, 다크 테마 기본, 모바일 375px 기준.
